Restart

几个月前使用过hexo 建过blog ， 当时还配置了inline-math ; 几个月后发现当时的math配置不能用了以及自己又忘了当时hexo 自己做的修改和指令。 现在只好重新建站，同时希望可以完成每周一篇post 发表。

What this post about ?

这次我会详细记录自己建站流程，以及遇到的坑和解决方式。

主要分为以下三块内容：

  • hexo 配置&主题设置
  • 我的需求如何实现
  • 日常使用

hexo 配置&主题设置

网上很多hexo 配置文章了，基本上就是：

  • 用npm 安装下hexo-cli
  • git clone next theme 指南这里有

在这里主要提下几个容易混淆的地方:

  • theme 安装： 需要在你起blog 的主要目录，然后git clone https://github.com/iissnan/hexo-theme-next themes/next
  • 两个很容易混淆的config：
    • 一个是在主目录下有个_config.yml : 这个是hexo的配置文件。主要设置站点名字，语言、作者、部署的网站,math(如果需要)
    • 然后在主目录/themes/next 下也有个_config.yml ：这个是theme 的配置文件。主要用来设置tag, category 和enable math 开关。

我的两个需求

建站当时主要有两个需求： 一个用来记录读过的论文，一个用来记录体重管理。

  • 论文的话，肯定离不开会有公式，google了hexo对math的支持，大多数是说hexo-math。 但其实hexo-math 这个包已经没在维护了，最后定位到网上有大佬做了介绍, 讲的很好，移步这里.
  • 体重管理需要一个在线交互的折线图: 这方面第一直觉想到的是echarts , 还好百度图说可以在线编辑图片然后导出代码块。但另外一个问题来了，hexo 可以渲染echarts 吗？ 搜了一波插件发现有人也实现了，直接上链接。

日常使用

hexo 很好用，此次几个月没用自己都忘了post 和page 有啥区别了，折腾了很久也遇到多次render 错误。这里总结下：

  • post: 就是一篇帖子，你发的一个博文就是一个post
  • page： 就是一个个你可以切换的页面，我觉得可以理解为主题

在路径显示上是这样的

布局 路径
post source/_posts
page source

说post 和page的作用是日常使用应该是基本上会以hexo new post “post_name” 为主，然后去source/_posts 下找到对应.md 文件进行修改。

几个日常操作:

  • hexo g (生成)
  • hexo s (本地起一个server， 可以用来查看)
  • hexo d （push 到github, 所有人都可以通过yourgitusername.github.io查看)
